# India Payment Pulse

## Project Overview
India Payment Pulse is an advanced analytics repository for simulated UPI transaction processing, data quality validation, ETL orchestration, PostgreSQL ingestion, and analytics-ready reporting.

## What this repository includes
- Simulated UPI transaction generation with dirty data patterns
- Extract, transform, and load pipeline for raw transaction data
- PostgreSQL ingestion with environment-driven configuration
- Parquet backup support for fast analytics reads
- Placeholder structure for dashboards and SQL analytics
- Data quality checks, schema validation, and text normalization

## Folder Structure
- `data/raw/` - Raw transaction CSV files generated by the simulator
- `data/clean/` - Cleaned data output and analytics-ready files
- `data/reports/` - Generated reports and summaries
- `scripts/` - ETL scripts and pipeline orchestration
- `dashboard/` - Dashboard or visualization application files
- `sql/` - SQL scripts, schema DDL, and analytics queries
- `logs/` - Pipeline logs and diagnostics

## Requirements
- Python 3.10+
- PostgreSQL 14+ (or compatible database)
- `pip install -r requirements.txt`

## Setup
1. Create and activate a virtual environment:
   ```powershell
   python -m venv venv
   .\venv\Scripts\Activate.ps1
   ```
2. Install Python dependencies:
   ```powershell
   pip install -r requirements.txt
   ```
3. Copy the example environment file and configure your database URL:
   ```powershell
   copy .env.example .env
   ```

## Configuration
- `DATABASE_URL` should point to your PostgreSQL instance.
- Default fallback is `postgresql://postgres:7280@localhost:5432/payment_pulse`.

## Running the Project
1. Generate raw transaction data:
   ```powershell
   python scripts/generate_data.py
   ```
2. Validate raw data extraction:
   ```powershell
   python scripts/extract.py
   ```
3. Clean and transform raw data:
   ```powershell
   python scripts/transform.py
   ```
4. Run the full ETL pipeline:
   ```powershell
   python scripts/load.py
   ```

## Data Schema
The transaction dataset contains:
- `txn_id`
- `timestamp`
- `sender_bank`
- `receiver_bank`
- `upi_app`
- `amount`
- `status`
- `merchant`
- `city`
- `category`

Cleaned data also includes:
- `processing_fee`
- `hour`

## Notes
- `data/raw/` is configured as input data.
- `data/clean/` is the target for analytics-ready output.
- `dashboard/` and `sql/` are ready for future visualization and analytics work.
- `.env` is ignored by Git to keep secrets safe.
